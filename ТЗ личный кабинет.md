# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## на разработку модулей «Личный кабинет» и «Специализация инженера»

## 1. ОБЩИЕ СВЕДЕНИЯ

Настоящее техническое задание описывает интеграцию двух ключевых подсистем приложения:
1. **Модуль «Специализация»** — бизнес-логика, определяющая профиль допуска инженера к конкретным видам оборудования.
2. **Модуль «Личный кабинет» (ЛК)** — пользовательский интерфейс для управления профилем, настройками, избранными объектами и получения сводной информации.

Эти модули тесно связаны: Личный кабинет является основным интерфейсом, через который Инженер и Территориальный менеджер (ТМ) просматривают и изменяют специализацию, а также используют её для фильтрации контента (например, списка Избранных объектов).

---

## 2. МОДУЛЬ «СПЕЦИАЛИЗАЦИЯ ИНЖЕНЕРА»

### 2.1. Бизнес-правила и матрица доступа
Каждый пользователь с ролью **Инженер** имеет характеристику «Специализация», ограничивающую виды оборудования, к которым он имеет доступ.

| Значение специализации | Доступные виды оборудования (коды) | Кол-во видов |
| :--- | :--- | :---: |
| **Только ВиК** *(Вентиляция и Кондиционирование)* | `vent`, `vrv_vn`, `vrv_nar`, `mssvn`, `mssnar`, `splitvn`, `splitnar` | 7 |
| **Только ИСЖ** *(Инженерные Сети и Электрика)* | `rsch`, `schetchik_gvs`, `schetchik_hvs`, `schetchik_electroshc`, `seti_vodosnab`, `teplovye_seti` | 6 |
| **ВиК + ИСЖ** | Все 13 видов оборудования | 13 |

**Правила системы:**
1. **Значение по умолчанию:** `ИСЖ`. Присваивается автоматически при создании инженера, если специализация не указана явно.
2. **Блокировка профиля:** Если у инженера не выбрана ни одна специализация, при входе в PWA отображается блокирующий экран с требованием выбрать специализацию. Создание визитов недоступно.
3. **Фильтрация и скрытие:** Инженер не видит в списке визитов задачи, не соответствующие его специализации. При создании визита на объекте, где нет профильного оборудования, кнопка «Создать визит» блокируется.
4. **Активные визиты:** Изменение специализации не отбирает у инженера уже активные визиты (статусы «В работе», «Завершен»).
5. **Отображение в отчетах:** Специализация инженера выводится в шапке PDF-акта выполненных работ.

### 2.2. Управление специализацией
* **Инженер:** Может самостоятельно выбрать или изменить свою специализацию в Личном кабинете.
* **ТМ / Администратор:** Может назначать и изменять специализацию подчиненного инженера через карточку пользователя или массовый импорт CSV.

## 3. МОДУЛЬ «ЛИЧНЫЙ КАБИНЕТ» (ЛК)

Личный кабинет предоставляет пользователям доступ к персональным настройкам, функции «Избранное» и оперативной сводке.

### 3.1. Личный кабинет Инженера (Mobile First, PWA)
*Доступ:* иконка профиля в нижней/верхней панели PWA. Открывается как выдвижная панель (Drawer).

**Функциональные блоки:**
1. **Профиль и безопасность:**
   * Отображение ФИО, должности, имени ТМ.
   * **Блок «Специализация»:** Чекбоксы `ВиК` и `ИСЖ`. Инженер может менять их самостоятельно. Изменения сохраняются на сервер и синхронизируются.
   * Кнопка «Сменить пароль».
   * Индикатор статуса сети: 🟢 «В сети» / 🔴 «Офлайн».
2. **Избранные объекты (⭐):**
   * Список объектов, добавленных в избранное (лимит до 20).
   * Отображение полного адреса и аналитического кода (`RU/77/123`).
   * Кнопка «+ Создать визит» напротив каждого объекта.
   * *Интеграция со специализацией:* Если на избранном объекте нет оборудования, соответствующего текущей специализации инженера, кнопка «Создать визит» неактивна, а рядом отображается иконка-подсказка ️ «Нет профильного оборудования».
   * **Офлайн-кэш:** Список хранится в IndexedDB. В режиме «Авиа» инженер видит избранное и может создать черновик визита.
3. **Быстрая статистика:** Сводка за текущий месяц (кол-во визитов, кол-во выявленных неисправностей).

### 3.2. Личный кабинет Территориального менеджера (Desktop First)
*Доступ:* выпадающее меню в шапке Web-интерфейса.

**Функциональные блоки:**
1. **Профиль:** ФИО, email, смена пароля.
2. **Управление командой и специализацией:**
   * Краткий список подчиненных инженеров с их текущим статусом.
   * Возможность быстро перейти в карточку инженера и изменить его **Специализацию** (ВиК / ИСЖ).
3. **Избранные объекты с индикацией:**
   * Список объектов с цветовыми маркерами.
   * 🔴 Красный кружок, если на объекте есть визит со статусом «Неисправно» или «Просрочено».
4. **Центр уведомлений:** Лента событий (завершение визитов, критические неисправности).

### 3.3. Личный кабинет Администратора
*Доступ:* выпадающее меню в шапке Web-интерфейса.
* **Функции:** Настройки безопасности (2FA), системные логи (история входов, история массовых импортов CSV), быстрые ссылки на справочники.

## 4. ТРЕБОВАНИЯ К ДАННЫМ И АРХИТЕКТУРЕ

### 4.1. Изменения в таблице `users`
Добавляются поля для хранения специализации:

| Поле | Тип | Описание | Значение по умолчанию |
| :--- | :--- | :--- | :--- |
| `specialization_vik` | BOOLEAN | Допуск к ВиК | `FALSE` |
| `specialization_iszh` | BOOLEAN | Допуск к ИСЖ | `TRUE` |

*Ограничение (CHECK):* `(specialization_vik = TRUE OR specialization_iszh = TRUE)`

### 4.2. Новая таблица `user_favorite_objects` (Избранное)

| Поле | Тип | Описание | Ограничения |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Первичный ключ | PK, NOT NULL |
| `user_id` | UUID | ID пользователя (Инженер или ТМ) | FK -> `users.id` |
| `object_code` | VARCHAR(50) | Аналитический код объекта | FK -> `addresses.object_code` |
| `added_at` | TIMESTAMP | Дата добавления | DEFAULT CURRENT_TIMESTAMP |

*Индексы:* Уникальный составной индекс `UNIQUE (user_id, object_code)`.

### 4.3. API Эндпоинты
* `GET /api/v1/profile` — получение профиля, включая поля специализации.
* `PATCH /api/v1/profile/specialization` — изменение специализации (доступно Инженеру, ТМ, Админу).
* `GET /api/v1/favorites` — получение списка избранного (с учетом офлайн-кэша для PWA).
* `POST /api/v1/favorites` — добавление в избранное.
* `DELETE /api/v1/favorites/{object_code}` — удаление из избранного.

## 6. КРИТЕРИИ ПРИЕМКИ (Acceptance Criteria)

### 6.1. Специализация
1. **Значение по умолчанию:** При создании инженера без явного указания специализации, в БД сохраняется `specialization_iszh = TRUE`.
2. **Самостоятельное изменение:** Инженер заходит в ЛК, снимает галочку `ИСЖ` и ставит `ВиК`. Изменения сохраняются, при следующем обновлении списка оборудования в визите отображаются только блоки ВиК.
3. **Блокировка пустого профиля:** Если вручную обнулить оба поля в БД, инженер при входе в PWA видит блокирующий экран с требованием выбрать специализацию.
4. **PDF-отчет:** В сгенерированном PDF в блоке «Общие сведения» присутствует строка «Специализация: ВиК / ИСЖ / ВиК + ИСЖ».

### 6.2. Личный кабинет и Избранное
5. **Офлайн-доступ:** Инженер добавляет объект в избранное при наличии сети. Включает режим «Авиа». Открывает ЛК — объект отображается. Нажимает «Создать визит» — визит создается и сохраняется в локальную очередь (IndexedDB).
6. **Интеграция с фильтрацией:** Инженер со специализацией `ВиК` добавляет в избранное объект, на котором есть только счетчики (ИСЖ). В ЛК рядом с объектом отображается иконка ⚠️, кнопка «Создать визит» неактивна.
7. **Индикация для ТМ:** ТМ добавляет объект в избранное. Инженер создает на этом объекте визит и ставит статус «Неисправно». В ЛК ТМ рядом с избранным объектом появляется красный кружок 🔴.
8. **Защита от дублей:** Попытка добавить один и тот же объект в избранное дважды игнорируется системой (возврат 409 Conflict или молчаливое обновление `added_at`).
